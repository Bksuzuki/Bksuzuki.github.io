Briefly describe your system (e.g. A store selling Pokemon game cards)

My website, The Countdown, is an online ecommerce website that sells, watches, suits, and glasses. It originally started with just watches and carried onto suits and glasses because I liked the kind of higher end feel.


Any notable shortcomings, bugs, problems, or additional features not implemented?

I definitely did have shortcomings. I didn't get my individual requirement done, which was the rating system. This was because I kept running into errors when trying to pass the ratings to the product information. In addition, I also didn't get the mailing system to work because the nodemailer module for some reason could not be found. I thiink if I had been in class, I would've been able to complete this with the class so it was unfortunate that I was sick. Another feature that I couldn't get working was the delete items in cart portion. However, you can edit that on the product pages.


Describe what you are most proud of about your system:

Although the delete protion does not work, I'm probably most proud of the cart. It took me so long to figure out how to display the cart and also invoice because I kept trying to make a convenient code block that would gather the information from all three arrays. However, in the end, I just ended up creating three code blocks in one table for the three kinds of products. This took very long to realize that I should just brute force it.


Describe what you are least happy with your system:

I'm least happy with the overall design. CSS was always the last thing on my mind because I wanted the website to work rather than looking good and adding extra things to beautify my website. I would definitely look into creating a more eye friendly website one day. In addition, I hate that the delete button does not work. I tried hard to get that to work but the undefined error kept on coming up.


How was developing this assignment different than assignment #2?

For assignment 2, there was a lot of code that I figured out on my own. In other words, I used syntax and functions that I personally made and understood. However, in Assignment 3, there were things such as nodemailer and the loadJSON function that I didn't understand but had to use. It got a little comlicated and I was definitely more confused. In addition, I had to rewrite so much stuff that the server.js file became pretty unorganized.


When you ran into a problem, what did you do to address it?

I struggled with a lot of problems for assignment 3. Errors constantly occured and things that were figured out in assignment 2 no longer worked. For the problems that I fixed, I just went line by line and tested for errors and tried to figure things out. Lots of mistakes were just small things and also mistakes in the order of things.


Describe what worked well in doing this assignment?

What worked well for me was actually looking at other code and trying to understand it. Although this took up a lot of time and I wasn't necessarily able to implement all of it, everytime that I gained understanding, I enjoyed and was able to use it.


Describe what did not work well in doing this assignment?

There was a lot of code in the examples that I tried to use but it didn't end up fitting with my assignemnt. I think really understanding the code and making up my own code blocks turned out to be more productive rather than attempting to copy and piece together an assignment.

I learned a lot about sessions and how helpful they are. Beforehand, it was required to pass all the information through the parameters but using the loadJSON function and the sessions was quite enlightening.

I would definitely just spend more time on my individual requirement. I was having such difficulty converting the stars selected into quantities and sending them to the JSON file so I eventually gave up and focused on the other aspects of the website.
